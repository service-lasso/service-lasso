# Project Template

Use this file to capture the project-specific intent that governance cannot provide.

## Purpose
Explain why this system exists and what problem it solves.

## Context
Describe the domain, users, and surrounding systems this project depends on.

## Constraints
List the constraints that shape delivery or behavior (technical, legal, security, budget, time).

## Risks
Call out the areas most likely to fail, be misused, or change unexpectedly.

## Assumptions
Make any implicit assumptions explicit so they can be revisited.

## Key Behaviors
Describe the core behaviors that must remain true over time.

## Verification Expectations
Describe how the system should be verified (tests, reviews, demos, or checks).
